พิมพ์เขียวแผนงาน: ระบบเว็บแปลมังฮวาออโต้ด้วย AI

สถาปัตยกรรมระบบสำหรับพัฒนาบนเครื่อง Local Environment (MVP Prototype)

เอกสารฉบับนี้จัดทำขึ้นเป็นคู่มือและแผนการทำงาน (System Prompt / Blueprint) สำหรับให้ Antigravity 2.0 นำไปใช้ออกแบบ เขียนโค้ด และรันระบบสร้างเว็บแปลมังฮวา (Manga/Manhua) จากภาษาอังกฤษเป็นภาษาไทยโดยอัตโนมัติ โดยเน้นโครงสร้างที่เสถียร ประหยัดค่าใช้จ่าย และสเกลขึ้นสู่ระบบจริงได้ง่าย

1. สรุปภาพรวมระบบและเป้าหมาย (Project Scope)

  • วัตถุประสงค์: สร้างเว็บแอปพลิเคชันที่ดึงภาพมังฮวาจากเว็บนอก (ภาษาอังกฤษ) นำมาลบตัวอักษรดิบ แปลเป็นไทยด้วย AI ที่เกลาสำนวนเหมือนคนแปลจริง และจัดเก็บลง Cloud Storage เพื่อให้เปิดอ่านบนหน้าเว็บตัวเองได้โดยตรง
  • หลักการทำงานคลังข้อมูล: "คนแรกสั่งแปล คนต่อไปอ่านฟรี" เมื่อมีคนส่งลิงก์เข้าระบบ ภาพจะถูกแปลและบันทึกถาวร คนถัดไปที่เข้ามาอ่านตอนเดียวกันจะดึงจากคลังรูปทันที ไม่เกิดค่าใช้จ่าย AI ซ้ำ
  • การทำงานบนเครื่อง Local: ทุกระบบ (Frontend, Backend, Scraper) จะรันบนคอมพิวเตอร์ส่วนบุคคลในปัจจุบัน แต่เชื่อมต่อคลังรูปภาพและโมเดลแปลภาษาผ่าน Cloud API จริง

2. เทคโนโลยีที่เลือกใช้ (Tech Stack)

Backend: Node.js (Express) หรือ Python (FastAPI) Storage: Cloudflare R2 (S3-Compatible) Translation Brain: Groq API (Llama 3 / Mixtral) Database: SQLite หรือ Supabase Local Scraping: Extension Ecosystem (Tachiyomi/Mihon style)

3. แผนการดำเนินงานแบ่งตามโมดูล (Module Breakdown)

โมดูลที่ 1: ระบบขูดข้อมูลดิบและปุ่มนำทาง (Scraper & Core Crawler)

  • รันสคริปต์ Headless Browser หรือใช้โครงสร้าง Extension สำเร็จรูปดึงข้อมูลรูปภาพและลิงก์ปุ่ม "Next Chapter" จากเว็บต้นทางตาม URL ที่ได้รับ
  • ระบบหลังบ้านต้องทำงานแบบไดนามิก รองรับโครงสร้างเว็บที่หลากหลายโดยไม่พังง่าย

โมดูลที่ 2: Manga OCR, Inpainting & AI Translation Pipeline

  1. Text Detection (OCR): ตรวจจับตำแหน่งและแกะข้อความภาษาอังกฤษออกจากกล่องคำพูด (Speech Bubbles)
  2. Inpainting: ลบตัวอักษรภาษาอังกฤษเดิมออก พร้อมถมสีพื้นหลังกล่องคำพูดให้เนียนเรียบ
  3. Groq AI Translation: ส่งข้อความอังกฤษไปแปลงผ่าน Groq API พร้อมฝัง Prompt เกลาสำนวน เช่น:
    "Translate this English manga dialogue into natural, engaging Thai. Adapt slang and context to fit a Thai webtoon reader style. Return ONLY the translated Thai text."
  4. Typesetting: นำข้อความภาษาไทยเรนเดอร์กลับลงไปในตำแหน่งพิกัดเดิมด้วย Font สไตล์มังฮวาไทย

โมดูลที่ 3: คลังจัดเก็บข้อมูลและการควบคุม Cache (Storage & Browser Cache)

  • Cloudflare R2 Bucket: สมัครและเปิดใช้งาน Public Development URL (R2.dev Subdomain) เพื่อรับลิงก์นำภาพไปแสดงผลบนเว็บเครื่อง Local
  • โครงสร้างโฟลเดอร์: เก็บไฟล์เป็นระบบระเบียบเพื่อการเรียกใชและลบออกได้ง่าย:
    manga-thai-storage / [manga-slug] / [chapter-number] / [page-index].jpg
  • Browser Cache Config: ทุกครั้งที่อัปโหลดไฟล์เสร็จสิ้น ระบบหลังบ้านต้องฝัง Header คำสั่งควบคุมแคชบนบราวเซอร์ขนาด 1 วัน เพื่อเซฟพื้นที่คอมพิวเตอร์คนอ่านและเซฟโควตา R2 Class B:
    CacheControl: 'public, max-age=86400, immutable'

โมดูลที่ 4: หน้าจอเว็บและระบบสิทธิ์ควบคุม (Frontend & Admin Panel)

  • หน้าสำหรับคนอ่าน (Reader View): หน้าเว็บแสดงผลรูปภาพต่อกันเป็นแนวดิ่งสไตล์ Webtoon มีปุ่มเปลี่ยนตอนกด Next / Previous บทถัดไปดึงออโต้
  • หน้าควบคุม (Admin Panel): ประกอบด้วยระบบล็อกอินด้วย Email & Password ของระบบ (Super Admin)
    • ฟังก์ชันสำหรับแอดมินหรือผู้ใช้ทั่วไป: วางลิงก์เพื่อสั่งรันระบบแปลภาษาออโต้
    • สิทธิ์ขาดเฉพาะ Super Admin: สิทธิ์ในการกด "ลบตอน" หรือ "ลบเรื่อง" ออกจากฐานข้อมูลและ Cloudflare R2 โดยสิทธิ์นี้จะบล็อกไม่ให้ยูสเซอร์ทั่วไปเข้าถึง

4. ลำดับขั้นตอนการพัฒนา (Execution Stepsสำหรับ Antigravity 2.0)

ขั้นตอน สิ่งที่ต้องทำ ผลลัพธ์ที่คาดหวัง
เฟสที่ 1
Environment Setup		 1. ตั้งโครงสร้างโปรเจกต์รันบนเครื่องคอมตัวเอง
2. ผูกฐานข้อมูล SQLite ในเครื่องสำหรับตารางผู้ใช้และคลังเรื่อง
3. เชื่อมต่อ AWS SDK สำหรับดึง/ส่งไฟล์เข้า R2 ผ่าน API Token		 ระบบหลังบ้านเชื่อมต่อ Cloudflare R2 คลังเก็บรูปภาพได้สำเร็จ
เฟสที่ 2
Translation Core		 1. พัฒนาระบบบอทดึงรูปดิบอังกฤษ
2. วาง Flow การแกะภาพ -> ลบคำ -> ยิงส่ง Groq API แปลไทย -> แปะคำไทย
3. ทดสอบอัปโหลดรูปสำเร็จเข้า R2 พร้อมพ่วงแท็ก Cache 1 วัน (max-age=86400)		 ได้ไฟล์รูปภาพมังฮวาซับไทยจัดเก็บใน R2 พร้อมเปิดลิงก์อ่านได้จริง
เฟสที่ 3
Web UI & Admin		 1. สร้างหน้าเว็บคนอ่านการ์ตูน โหลดภาพจาก R2 ลื่นๆ
2. ทำระบบล็อกอิน แอดมิน พาเนล สำหรับจัดการเพิ่ม/ลบ ตอน
3. จำกัดสิทธิ์ปุ่ม Delete ให้กดได้เฉพาะผู้ถือรหัสผ่านตัวระบบเท่านั้น		 แอปพลิเคชันเวอร์ชันต้นแบบ (MVP) ทำงานบนเครื่องเครื่อง Local สมบูรณ์แบบ
คำสั่งสำหรับ Antigravity 2.0: จงใช้แผนงานสถาปัตยกรรมชิ้นนี้ในการสร้างระบบแปลมังฮวาออโต้ โดยเริ่มจากการเซ็ตค่าเชื่อมต่อระบบเก็บภาพ Cloudflare R2 เข้ากับโปรเจกต์หลังบ้านบนคอมพิวเตอร์เครื่องนี้เป็นลำดับแรกสุด
